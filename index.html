HTML

  • HTML is the frame of a web application
  • HTML stands for Hypertext Markup Language
  • A HTML attribute defines additional information about an element
  • HTML is made up of elements, some are seen on the webpage and some aren't
  • Commonly used elements are Text, Image, Header and List
  • Meta-related, Content devision and Semantic are all types of other elemetns used
  • There are 6 different heading sizes ranging from H1 to H6
  • Heading 1 should only be used on a page once
  • Elements are put inside angle brakcets
  • The closing tag has a / after the first angle bracket

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The section element helps oragnise the page
  • The li is element used to create bullet points
  • The ul is element used to create an unordered list
  • The ol is element used to create an ordered list
  • The p is element for a paragraph
  • The title element defines the title for a webpage
  • The a element (anchor) creats a hyperlink to
  • The br is element which create a break in the lines
  • The charset element sets our character encoding
  • The http-equiv element sets an HTTP header for content
  • The viewport element sets the default view for our webpage
  • The style element defines the CSS styles associated with a website
  • The base element defines the base URL for a webpage
  • The link element connects an external resource to the HTML document
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport
  • The main and the footer are both elements within the section element
  • The img element contains information about the image

  • The src attribute contains the loction of the image
  • The alt attribute is the image description
  • The lang="en" attribute tells the browser it is in english
  • Width and height attributes make the image size
  • Parent elements have child elements within them
  • !DOCTYPE html is a declaration telling the browser to expect HTML

CSS

  • CCS stands for Cascading Style Sheets
  • There are 3 styles of CCS: inline, internal and external
  • Class selectors apply CSS to HTML
  • The card selector can add a box with shadowing
  • Width, margin, border and box-shadow are used in a card selector
  • The width is in a percentage, the margin, border and box-shadow are in pixles
  • The border also displays a colour and the box shadow displays a colour number

    • Inline

  • Inline styles apply to specific HTML tags
  • Styles a particular page element using a style attribute with a CSS rule

    • Internal

  • Internal styles are placed in the head section of the HTML file
  • All tagged elements on the page will change

    • External

  • External styles use link element to the style file which are placed in the head
  • This is the most prefered style
  • Design changes are easier as CSS rules are in a seperate file

GIT

  • Git commands in the terminal start with git
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git push: sends updates branches on git
  • git pull: retrives the main or branches from guthub
  • git -m: to assocaite a message with the commit
  • git add -A: commits all files to the working branch

JAVASCRIPT

  • Javascript runs top to bottom
  • Javascript makes the application interactive
  • Javascript uses arrays and loops
  • Loops operate each element in an array
  • Curly brackets contain the code for the task
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
